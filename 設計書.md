### 要件定義書との設計差分

本設計書は要件定義書 v1.1 に準拠するが、以下の点で要件を具体化・拡張している。

| 項目 | 要件定義書 v1.1 | 本設計書 | 差分理由 |
|------|----------------|---------|---------|
| ステータス値 | 4.4.2節で `pending_approval` を含む9値を定義済み | TypeScript型 `DocumentStatus` で同一9値を採用（3.1.2節） | 差分なし（完全一致） |
| 帳票削除 | 管理者に削除権限あり（6.2節権限マトリクス） | `draft` ステータスの論理削除のみ許可に制限（4.1節） | 電子帳簿保存法準拠: 確定済みPDFはObject Lockで保護されるため、DBヘッダーのみ削除可能な状態を防止 |
| Server Actions / Queries | MCPツールと画面機能を定義（5章・10章） | `src/actions/` `src/queries/` として配置を具体化（2章・4章） | Next.js App Router の実装パターンへの落とし込み |

### 1.2 リクエストフロー

#### Web UI 経由

```
Browser → Next.js Middleware (JWT検証)
       → Server Component / Server Action
       → Service Layer
       → DynamoDB / S3
       → Response (RSC Payload / JSON)
```

#### MCP 経由

```
Claude/Slack → MCP Server (OAuth 2.0 トークン検証)
            → Tool Handler (バリデーション + 冪等性チェック)
            → Service Layer（Web UIと共通）
            → DynamoDB / S3
            → MCP Response (JSON)
```

### 1.3 設計原則

| 原則 | 適用 |
|------|------|
| サービス層の共通化 | Web UI と MCP は同一の Service Layer を呼び出す。ビジネスロジックの二重実装を防止 |
| 四眼原則 | 全承認操作で作成者 ≠ 承認者をシステムで強制 |
| 冪等性 | 作成系操作は Idempotency-Key で二重実行を防止 |
| 監査証跡 | 全操作を AuditLogService 経由で記録。サービス層のデコレータで自動化 |
| 電子帳簿保存法準拠 | 確定済み PDF は S3 Object Lock で改ざん防止。取消時は派生保存 |

#### 7.1.1 削除制約

帳票削除は **電子帳簿保存法との整合性** を確保するため、以下の制約を設ける。

| 操作 | 対象ステータス | 動作 | 理由 |
|------|--------------|------|------|
| 論理削除（`isDeleted: true`） | `draft` のみ | DB上の `isDeleted` フラグを `true` に更新。一覧から非表示化 | 下書きはPDF未生成のため法定保存義務なし |
| 物理削除 | 不可 | 全ステータスで物理削除を禁止 | 監査証跡の保全 |
| 確定以降の削除 | `pending_approval` / `confirmed` / `sent` / `paid` / `overdue` / `won` / `lost` / `cancelled` | `STATUS_CONSTRAINT_ERROR` を返却 | 確定済みPDFはS3 Object Lockで保護されており、DBヘッダーのみ削除される不整合を防止 |

```typescript
// services/document.service.ts (deleteDocument)

async function deleteDocument(documentId: string, userId: string): Promise<void> {
  const doc = await documentRepo.get(documentId);
  if (!doc) throw new NotFoundError('DOCUMENT_NOT_FOUND');
  if (doc.status !== 'draft') {
    throw new StatusConstraintError(
      'STATUS_CONSTRAINT_ERROR',
      'Only draft documents can be deleted'
    );
  }
  await documentRepo.update(documentId, { isDeleted: true, updatedBy: userId });
  await auditLogService.log('DOCUMENT_DELETED', documentId, userId);
}
```

### 7.2 消費税計算ロジック

```typescript
// lib/tax-calculator.ts

interface TaxSummary {
  subtotal: number;           // 税抜合計
  tax10Subtotal: number;      // 10%対象小計
  tax10Amount: number;        // 10%消費税額
  tax8Subtotal: number;       // 8%対象小計
  tax8Amount: number;         // 8%消費税額
  taxExemptSubtotal: number;  // 非課税小計
  taxAmount: number;          // 消費税合計
  totalAmount: number;        // 税込合計
}

function calculateTax(items: LineItem[]): TaxSummary {
  // 1. 税率ごとにグループ化
  const groups = groupBy(items, item => item.taxRate);

  // 2. 各グループの小計を算出
  const tax10Subtotal = sumAmount(groups[0.10] ?? []);
  const tax8Subtotal  = sumAmount(groups[0.08] ?? []);
  const exemptSubtotal = sumAmount(groups[0] ?? []);

  // 3. 消費税額を計算（税率区分ごとの合計に対して切り捨て）
  const tax10Amount = Math.floor(tax10Subtotal * 0.10);
  const tax8Amount  = Math.floor(tax8Subtotal * 0.08);

  const subtotal = tax10Subtotal + tax8Subtotal + exemptSubtotal;
  const taxAmount = tax10Amount + tax8Amount;

  return {
    subtotal,
    tax10Subtotal, tax10Amount,
    tax8Subtotal, tax8Amount,
    taxExemptSubtotal: exemptSubtotal,
    taxAmount,
    totalAmount: subtotal + taxAmount,
  };
}
```

async function nextDocumentNumber(
  type: 'estimate' | 'invoice',
  date: string  // YYYYMMDD
): Promise<string> {
  const prefix = type === 'estimate' ? 'EST' : 'INV';
  const pk = `SEQ#${type}#${date}`;

  // DynamoDB UpdateItem + ADD + ConditionExpression で排他制御
  const result = await dynamoClient.update({
    TableName: 'Sequences',
    Key: { PK: pk, SK: 'META' },
    UpdateExpression: 'ADD currentNumber :inc SET updatedAt = :now',
    ExpressionAttributeValues: {
      ':inc': 1,
      ':now': new Date().toISOString(),
    },
    ReturnValues: 'ALL_NEW',
  });

  const num = String(result.Attributes.currentNumber).padStart(3, '0');
  return `${prefix}-${date}-${num}`;
}

### 7.4 承認フロー（四眼原則）

```typescript
// services/approval.service.ts

async function approveDocument(
  documentId: string,
  approverId: string,
  action: 'approve' | 'reject',
  comment?: string
): Promise<void> {
  const doc = await documentRepository.get(documentId);
  const approver = await userRepository.get(approverId);

  // 四眼原則チェック
  if (doc.createdBy === approverId) {
    throw new PermissionError('自己承認は許可されていません（四眼原則）');
  }

  // 権限チェック
  if (!['accountant', 'admin'].includes(approver.role)) {
    throw new PermissionError('承認権限がありません');
  }

  // ステータスチェック
  if (doc.status !== 'pending_approval') {
    throw new InvalidStatusError(`現在のステータス: ${doc.status}`);
  }

  const newStatus = action === 'approve' ? 'confirmed' : 'draft';

  // トランザクション: ステータス更新 + 承認履歴追加
  await dynamoClient.transactWrite({
    TransactItems: [
      { Update: { /* status → newStatus */ } },
      { Put: { /* ApprovalRecord */ } },
    ],
  });

  // 承認時: PDF生成
  if (action === 'approve') {
    await pdfService.generate(documentId);
  }

  // 通知
  await notificationService.sendApprovalResult(doc, action, comment);

  // 監査ログ
  await auditLogService.log({ action: `document:${action}`, ... });
}
```

### 10.2 Slack通知

```typescript
// services/notification.service.ts

interface SlackMessage {
  channel: string;
  blocks: SlackBlock[];
}

// 承認依頼通知の例
function buildApprovalRequestMessage(doc: Document, requester: User): SlackMessage {
  return {
    channel: config.slackApprovalChannel,
    blocks: [
      {
        type: 'header',
        text: { type: 'plain_text', text: '📋 承認依頼' }
      },
      {
        type: 'section',
        fields: [
          { type: 'mrkdwn', text: `*帳票番号:* ${doc.documentNumber}` },
          { type: 'mrkdwn', text: `*種別:* ${doc.documentType === 'estimate' ? '見積書' : '請求書'}` },
          { type: 'mrkdwn', text: `*取引先:* ${doc.clientName}` },
          { type: 'mrkdwn', text: `*金額:* ¥${doc.totalAmount.toLocaleString()}` },
          { type: 'mrkdwn', text: `*依頼者:* ${requester.displayName}` },
        ]
      },
      {
        type: 'actions',
        elements: [
          { type: 'button', text: '承認画面を開く', url: `${baseUrl}/approvals` },
        ]
      }
    ]
  };
}
```
